# Диаграмма случаев использования, описывающая пользователей и случаи использования разрабатываемого приложения

## Обзор
Диаграмма показывает взаимодействие семи пользователей системы с основными функциями для управления дефектами на конвейере автомобильного завода.

## Акторы и их функции

**Ремонтник** - выполняет четыре основные операции:
- Зарегистрироваться как доступный
- Зарегистрировать дефект (включает указание причины и места на схеме)
- Зафиксировать начало ремонта
- Зафиксировать окончание ремонта

**Бригадир** - координирует работу ремонтников:
- Назначить ремонт рабочему
- Получить отчёт по работе бригады
- Может участвовать в регистрации дефектов

**Диспетчер** - управляет потоком автомобилей:
- Просмотреть статус ремонтных зон
- Направить автомобиль в ремонтную зону

**Отдел контроля качества** - получает:
- Сводный отчёт по смене (количество ремонтов, статистика по дефектам, среднее время)

**Начальник цеха** - получает:
- Отчёт по количеству и времени ремонтов

**Бухгалтерия** - получает:
- Отчёт по отработанным сменам каждого рабочего

**Администратор системы** - управляет справочными данными:
- Управлять данными ремонтных зон
- Управлять данными ремонтных мест
- Управлять данными ремонтных бригад

## Ключевые связи
Включаемые операции (include) - при регистрации дефекта обязательно указываются причина дефекта и его место на схеме автомобиля. Остальные связи показывают прямые взаимодействия акторов с функциями системы.
